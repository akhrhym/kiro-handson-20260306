🎯 このハンズオンで学ぶこと

このハンズオンでは、Kiro IDEの4つの主要機能を使って、実際にTODOアプリを作ります。
所要時間は 30〜45分 です。プログラミング経験がなくても大丈夫です。

📋 Specs（スペック）

自然言語から要件・設計・タスクを自動生成

→ 「こんなアプリ作りたい」と伝えるだけで設計書ができる

🎛️ Steering（ステアリング）

プロジェクト固有のルールをAIに伝達

→ 「TypeScriptを使って」「日本語コメントで」などのルール設定

💬 Chat（チャット）

コンテキストを理解したAIとの対話

→ プロジェクトの中身を理解した上で質問に答えてくれる

🔗 Hooks（フック）

ファイル変更時の自動処理

→ ファイルを保存したら自動でテストを実行、など

📖 はじめに知っておきたい用語

ハンズオンを始める前に、出てくる用語を確認しておきましょう。

IDE（統合開発環境）

コードを書く・実行する・デバッグするなど、開発に必要な機能がまとまったアプリケーションのこと。Kiroはこの一種です。

🏠 たとえるなら：料理に必要な道具が全部揃ったキッチン
Specs（仕様書）

アプリの「何を作るか」「どう作るか」「何をやるか」を文書化したもの。Kiroでは自然言語で伝えるだけで自動生成されます。

📐 たとえるなら：家を建てる前の設計図
Steering（ステアリング）

AIに対して「このプロジェクトではこういうルールで書いてね」と伝える設定ファイル。

🧭 たとえるなら：料理のレシピに書かれた「塩は控えめに」などの注意書き
Hooks（フック）

特定のイベント（ファイル保存など）が起きたときに、自動で処理を実行する仕組み。

🔔 たとえるなら：ドアが開いたら自動で電気がつく仕組み
React / TypeScript / Vite

今回使う技術スタック。ReactはUI構築ライブラリ、TypeScriptは型安全なJavaScript、Viteは高速な開発サーバーです。

🧱 たとえるなら：家を建てるための木材・釘・工具のセット
ローカルストレージ

ブラウザにデータを保存する仕組み。ページをリロードしてもデータが消えません。

🗄️ たとえるなら：ブラウザの中にある小さな引き出し

📋 前提条件

ハンズオンを始める前に、以下が準備できていることを確認してください。

項目 要件 確認方法
Kiro IDE インストール済み kiro.dev/downloads からダウンロード
Node.js v18以上 ターミナルで node -v を実行
アカウント GitHub / Google / AWS Builder ID のいずれか サインインできることを確認

Node.js はJavaScriptをパソコン上で動かすためのソフトウェアです。 まだインストールしていない場合は nodejs.org からLTS版をダウンロードしてください。

Step 1: プロジェクトのセットアップ ⏱ 約3分
1

Kiro IDEを起動してサインイン

Kiro IDEを起動し、GitHub、Google、またはAWS Builder IDでサインインします。

初回起動時にサインイン画面が表示されます。お持ちのアカウントでサインインしてください。どのアカウントでも機能は同じです。

2

新規プロジェクトフォルダを作成

File → Open Folder から空のフォルダを選択（または新規作成）

ターミナルで以下のコマンドを実行してフォルダを作成することもできます：

mkdir todo-app cd todo-app

mkdir は「フォルダを作る」コマンド、cd は「そのフォルダに移動する」コマンドです。 作成したフォルダをKiroの File → Open Folder で開いてください。

Step 2: Specsで仕様を作成 ⏱ 約5分

通常のアプリ開発では、まず「何を作るか」を決めてから「どう作るか」を設計し、「やることリスト」を作ります。 Kiroの Specs はこの3つを、あなたの説明から自動で生成してくれます。

3

Specの作成を開始

左サイドバーの 「SPECS」 セクションで 「+」 ボタンをクリック

Kiroの画面左側にあるサイドバーを見てください。「SPECS」という項目があります。その横の「+」ボタンを押すと、新しい仕様書の作成が始まります。

4

プロンプトを入力

以下のテキストをそのままコピーして貼り付けてください：

📝 入力するプロンプト
シンプルなTODOアプリを作成したい。
- タスクの追加・削除・完了切り替えができる
- React + TypeScriptで実装
- ローカルストレージでデータ永続化
- シンプルでモダンなUI

プロンプトが短すぎると、Kiroが意図を正しく理解できないことがあります。上記のように箇条書きで具体的に書くのがコツです。

5

自動生成されるファイルを確認

Kiroが以下の3つのファイルを自動で生成します。それぞれの役割を理解しましょう：

📋
requirements.md（要件定義）

「何を作るか」を定義したファイル

→ ユーザーストーリー（「ユーザーとして〇〇したい」）の形式で書かれています

🏗️
design.md（技術設計）

「どう作るか」を設計したファイル

→ 使う技術、ファイル構成、データの流れなどが書かれています

✅
tasks.md（タスクリスト）

「やることリスト」を整理したファイル

→ 実装の順番がチェックリスト形式で並んでいます

💡
各ファイルの内容を確認し、問題なければ 「Approve」 ボタンで承認してください。修正したい場合はチャットで指示できます。
Step 3: Steeringを設定 ⏱ 約3分

AIは万能ですが、プロジェクトごとの「お約束」は知りません。 たとえば「TypeScriptを使う」「日本語でコメントを書く」「anyは使わない」といったルールを Steering で伝えることで、 AIが生成するコードの品質が格段に上がります。

6

Steeringファイルを作成

左サイドバーの 「AGENT STEERING」 セクションで 「+」 ボタンをクリック

7

プロジェクトルールを記述

product.md を選択し、以下の内容をコピーして貼り付けてください：

# TODOアプリ プロジェクトガイドライン ## 技術スタック - React 18 - TypeScript - Vite - CSS Modules ## コーディング規約 - 関数コンポーネントを使用 - カスタムフックで状態管理ロジックを分離 - 日本語コメントを推奨 ## ディレクトリ構成 src/ ├── components/ # UIコンポーネント ├── hooks/ # カスタムフック ├── types/ # 型定義 └── utils/ # ユーティリティ関数 ## 禁止事項 - anyの使用禁止 - classコンポーネント禁止

この設定により、Kiroは今後このプロジェクトでコードを生成するとき、上記のルールに従います。 たとえば any 型を使わない、関数コンポーネントだけを使う、といったことを自動で守ってくれます。

Step 4: タスクに従って実装 ⏱ 約10分

Step 2で生成された tasks.md には、アプリを完成させるための手順が順番に並んでいます。 Kiroはこのタスクリストに沿って、1つずつコードを自動生成してくれます。

8

タスクリストを開いて実行

生成された tasks.md を開き、Chatで以下を指示：

📝 入力するプロンプト
最初のタスクから順番に実行してください

または、タスクを選択して 「Start task」 をクリック

Kiroがタスクを1つずつ実行し、必要なファイルを自動で作成・編集していきます。 各タスクが完了するたびに、生成されたコードを確認できます。わからないことがあればChatで質問できます。

9

生成されるコードを確認

Kiroが生成する主なファイルの例を紹介します。実際の出力は異なる場合があります。

型定義（src/types/todo.ts）

export interface Todo { id: string; text: string; completed: boolean; createdAt: Date; }

interface は「データの形」を定義するTypeScriptの機能です。 ここでは「TODOには id, text, completed, createdAt がある」と宣言しています。 これにより、タイプミスや間違ったデータ型を使ったときにエラーで教えてくれます。

カスタムフック（src/hooks/useTodos.ts）

import { useState, useEffect } from 'react'; import { Todo } from '../types/todo'; export const useTodos = () => { const [todos, setTodos] = useState<Todo[]>(() => { const saved = localStorage.getItem('todos'); return saved ? JSON.parse(saved) : []; }); useEffect(() => { localStorage.setItem('todos', JSON.stringify(todos)); }, [todos]); const addTodo = (text: string) => { const newTodo: Todo = { id: Date.now().toString(), text, completed: false, createdAt: new Date() }; setTodos([...todos, newTodo]); }; const toggleTodo = (id: string) => { setTodos(todos.map(todo => todo.id === id ? { ...todo, completed: !todo.completed } : todo )); }; const deleteTodo = (id: string) => { setTodos(todos.filter(todo => todo.id !== id)); }; return { todos, addTodo, toggleTodo, deleteTodo }; };

カスタムフックは、Reactの状態管理ロジックを再利用可能な形にまとめたものです。 ここでは「TODOの追加・切り替え・削除」と「ローカルストレージへの保存」をひとまとめにしています。 UIのコードと分離することで、見通しがよくなります。

メインコンポーネント（src/App.tsx）

import { useState } from 'react'; import { useTodos } from './hooks/useTodos'; import './App.css'; function App() { const [inputValue, setInputValue] = useState(''); const { todos, addTodo, toggleTodo, deleteTodo } = useTodos(); const handleSubmit = (e: React.FormEvent) => { e.preventDefault(); if (inputValue.trim()) { addTodo(inputValue.trim()); setInputValue(''); } }; return ( <div className="app"> <h1>📝 TODOアプリ</h1> <form onSubmit={handleSubmit} className="todo-form"> <input type="text" value={inputValue} onChange={(e) => setInputValue(e.target.value)} placeholder="新しいタスクを入力..." className="todo-input" /> <button type="submit" className="add-btn">追加</button> </form> <ul className="todo-list"> {todos.map(todo => ( <li key={todo.id} className={todo.completed ? 'completed' : ''}> <input type="checkbox" checked={todo.completed} onChange={() => toggleTodo(todo.id)} /> <span>{todo.text}</span> <button onClick={() => deleteTodo(todo.id)}>🗑️</button> </li> ))} </ul> <p className="todo-count"> 残りタスク: {todos.filter(t => !t.completed).length}件 </p> </div> ); } export default App;

これがアプリの「見た目」を担当するメインコンポーネントです。 先ほどのカスタムフック useTodos を呼び出して、TODOデータと操作関数を受け取っています。 フォームの送信、チェックボックスの切り替え、削除ボタンのクリックがそれぞれの関数に紐づいています。

Step 5: Hooksを設定（オプション） ⏱ 約3分

Hooks を使うと、「ファイルを保存したら自動でテストを実行する」といった自動化ができます。 手動で毎回テストを実行する手間が省け、バグの早期発見にもつながります。

10

自動テスト生成Hookを作成

左サイドバーの 「AGENT HOOKS」 セクションで 「+」 ボタンをクリック

設定項目値
EventFile Saved
File pathsrc/**/*.{ts,tsx}
📝 Instructions
ファイルが保存されたら：
1. 変更内容を分析
2. 対応するテストファイルを作成・更新
3. テストを実行して結果を報告

src/**/*.{ts,tsx} は「srcフォルダ以下の全ての .ts と .tsx ファイル」を意味します。 このHookを設定すると、コードを保存するたびにKiroが自動でテストを作成・実行してくれます。

11

README自動更新Hookを作成（オプション）

設定項目値
EventFile Saved
File pathsrc/components/**/*.tsx
📝 Instructions
コンポーネントが変更されたら：
1. コンポーネントのPropsを分析
2. README.mdのコンポーネント一覧を更新
3. 使用例を追加
ℹ️
Hooksは自動で実行されるため、頻繁に変更するファイルに設定する場合は注意してください。
Step 6: 動作確認 ⏱ 約5分
12

依存関係をインストール

ターミナルで以下のコマンドを実行します：

npm install

npm install は、プロジェクトが必要とするライブラリ（React、TypeScriptなど）をダウンロードするコマンドです。 初回のみ実行が必要で、完了すると node_modules フォルダが作成されます。

13

開発サーバーを起動

npm run dev

ブラウザで http://localhost:5173 を開いて動作確認

npm run dev は開発用サーバーを起動するコマンドです。 コードを変更すると自動でブラウザに反映されます（ホットリロード）。 サーバーを停止するには、ターミナルで Ctrl + C を押します。

🔧 困ったときは

うまくいかないときは、以下を確認してみてください。

❌ Specsが生成されない

プロンプトが短すぎるか、具体性が不足している可能性があります。

解決方法：
  • より具体的な要件を記述する（箇条書きがおすすめ）
  • 技術スタック（React、TypeScriptなど）を明示する
  • インターネット接続を確認する
❌ npm run dev でエラーが出る

依存関係のインストールが完了していない可能性があります。

解決方法：
rm -rf node_modules npm install

↑ node_modulesフォルダを削除して、もう一度インストールし直すコマンドです

❌ TypeScriptエラーが発生する

型定義が不足しているか、インポートパスが間違っている可能性があります。

解決方法：
  • Chatで「TypeScriptエラーを修正して」と依頼する
  • tsconfig.json の設定を確認する
  • ファイルのインポートパス（相対パス）が正しいか確認する
❌ ページをリロードするとデータが消える

ローカルストレージへの保存処理が正しく動いていない可能性があります。

解決方法：
  • ブラウザの開発者ツール → Application → Local Storage で確認
  • シークレットモードでは保存されない場合があります
  • Chatで「ローカルストレージの保存が動いていません」と相談する
