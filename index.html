🎯 このハンズオンで学ぶこと

このハンズオンでは、Kiro IDEの4つの主要機能を使って、実際にTODOアプリを作ります。
所要時間は 30〜45分 です。プログラミング経験がなくても大丈夫です。
ブラウザだけで動くアプリを作ります。

📋 Specs（スペック）

自然言語から要件・設計・タスクを自動生成

→ 「こんなアプリ作りたい」と伝えるだけで設計書ができる

🎛️ Steering（ステアリング）

プロジェクト固有のルールをAIに伝達

→ 「HTMLファイルは1つにまとめて」「日本語コメントで」などのルール設定

💬 Chat（チャット）

コンテキストを理解したAIとの対話

→ プロジェクトの中身を理解した上で質問に答えてくれる

🔗 Hooks（フック）

ファイル変更時の自動処理

→ ファイルを保存したら自動でテストを実行、など

📖 はじめに知っておきたい用語

ハンズオンを始める前に、出てくる用語を確認しておきましょう。

IDE（統合開発環境）

コードを書く・実行する・デバッグするなど、開発に必要な機能がまとまったアプリケーションのこと。Kiroはこの一種です。

🏠 たとえるなら：家を建てるための工具や材料が全部揃った作業場
Specs（仕様書）

アプリの「何を作るか」「どう作るか」「何をやるか」を文書化したもの。Kiroでは自然言語で伝えるだけで自動生成されます。

📐 たとえるなら：家を建てる前の設計図
Steering（ステアリング）

AIに対して「このプロジェクトではこういうルールで書いてね」と伝える設定ファイル。

🧭 たとえるなら：家を建てるときの「窓は二重ガラスにする」「壁は白で統一」などの施工ルール
Hooks（フック）

特定のイベント（ファイル保存など）が起きたときに、自動で処理を実行する仕組み。

🔔 たとえるなら：家のドアが開いたら自動で電気がつくセンサーの仕組み
HTML / CSS / JavaScript

今回使う技術スタック。HTMLはページの構造、CSSは見た目のデザイン、JavaScriptは動きや処理を担当します。 この3つだけでWebアプリが作れます。

🧱 たとえるなら：HTMLが骨組み、CSSが壁紙やペンキ、JavaScriptが電気配線
ローカルストレージ

ブラウザにデータを保存する仕組み。ページをリロードしてもデータが消えません。

🗄️ たとえるなら：家の中にある小さな収納棚。しまったものは次に来たときもそのまま残っている

📋 前提条件

ハンズオンを始める前に、以下が準備できていることを確認してください。

項目 要件 確認方法
Kiro IDE インストール済み kiro.dev/downloads からダウンロード
ブラウザ Chrome / Edge / Firefox など 最新版を推奨
アカウント GitHub / Google / AWS Builder ID のいずれか サインインできることを確認

HTML・CSS・JavaScriptだけで動くので、ブラウザさえあればOKです。 ファイルをダブルクリックするだけでアプリが開きます。

Step 1: プロジェクトのセットアップ ⏱ 約3分
1

Kiro IDEを起動してサインイン

Kiro IDEを起動し、GitHub、Google、またはAWS Builder IDでサインインします。

初回起動時にサインイン画面が表示されます。お持ちのアカウントでサインインしてください。どのアカウントでも機能は同じです。

2

新規プロジェクトフォルダを作成

File → Open Folder から空のフォルダを選択（または新規作成）

ターミナルで以下のコマンドを実行してフォルダを作成することもできます：

mkdir todo-app cd todo-app

mkdir は「フォルダを作る」コマンド、cd は「そのフォルダに移動する」コマンドです。 作成したフォルダをKiroの File → Open Folder で開いてください。

Step 2: Specsで仕様を作成 ⏱ 約5分

通常のアプリ開発では、まず「何を作るか」を決めてから「どう作るか」を設計し、「やることリスト」を作ります。 Kiroの Specs はこの3つを、あなたの説明から自動で生成してくれます。

3

Specの作成を開始

画面右側のチャットパネルに 「Let's build」 画面が表示されます。 「Spec」 タブを選択してください。

「Spec」は「Plan first, then build. Create requirements and design before coding starts.」と説明されています。 設計してから作る、仕様駆動開発のアプローチです。

「Vibe」タブはチャットしながら自由に開発するモード、「Spec」タブは仕様を先に作ってから開発するモードです。 今回は「Spec」を選びます。左サイドバーの「SPECS」セクションの「+ Create New Spec」ボタンからも同様に作成できます。

4

プロンプトを入力

チャット入力欄に以下のテキストをコピーして貼り付け、送信してください：

シンプルなTODOアプリを作成したい。 - タスクの追加・削除・完了切り替えができる - HTML + CSS + JavaScript のみで実装（フレームワーク不要） - ローカルストレージでデータ永続化 - index.html 1ファイルに HTML・CSS・JS をまとめる - シンプルでモダンなUI

プロンプトが短すぎると、Kiroが意図を正しく理解できないことがあります。上記のように箇条書きで具体的に書くのがコツです。

Kiroがファイル操作やコマンドを実行する際、「Waiting on your input」と表示されることがあります。 これはKiroがあなたの許可を待っている状態です。内容を確認して 「Run」 または 「Trust」 をクリックしてください。 不要な操作の場合は「Reject」で拒否できます。

5

Specの種類を選択

プロンプトを送信すると、Kiroが「これは新機能の開発ですか？それともバグ修正ですか？」と聞いてきます。

今回は新しいアプリを作るので、「Build a Feature」（Recommended）を選択してください。

Kiroは「新機能の開発」と「バグ修正」で異なるワークフローを用意しています。 新機能の場合は要件定義→設計→タスクの流れ、バグ修正の場合はバグの原因調査から始まります。

6

ワークフローを選択

次に「何から始めますか？」と聞かれます。以下の2つから選べます：

  • Requirements（推奨）— 要件定義から始める
  • Technical Design — 技術設計から始める

今回は 「Requirements」（Recommended）を選択してください。

「何を作るか」を先に決めてから「どう作るか」を設計する流れが一般的です。 技術的なアプローチが先に決まっている場合は「Technical Design」を選ぶこともできます。

7

要件定義（requirements.md）を確認・承認

Kiroがまず requirements.md（要件定義）を生成します。 内容を確認し、問題なければチャットの 「Move to design phase」 ボタンをクリックしてください。

📋
requirements.md（要件定義）

「何を作るか」を定義したファイル

→ ユーザーストーリー（「ユーザーとして〇〇したい」）の形式で書かれています

💡
修正したい場合はチャットで「〇〇を追加して」「〇〇は不要」などと指示できます。 テキスト入力フォームに何も入力せずにそのまま進むと、現在の内容で承認されます。 納得できたら次に進みましょう。
8

技術設計（design.md）を確認・承認

要件定義を承認すると、次に design.md（技術設計）が生成されます。 同様に内容を確認し、問題なければ承認してください。

🏗️
design.md（技術設計）

「どう作るか」を設計したファイル

→ 使う技術、ファイル構成、データの流れなどが書かれています

設計ドキュメントの生成には少し時間がかかることがあります。 「Working...」と表示されている間はKiroが作業中です。

9

タスクリスト（tasks.md）を確認

設計を承認すると、最後に tasks.md（タスクリスト）が生成されます。

✅
tasks.md（タスクリスト）

「やることリスト」を整理したファイル

→ 実装の順番がチェックリスト形式で並んでいます

💡
Specの作成は段階的に進みます。各ドキュメントの内容を確認してから次に進むので、 途中で方向修正がしやすいのがポイントです。
Step 3: Steeringを設定 ⏱ 約3分

AIは万能ですが、プロジェクトごとの「お約束」は知りません。 たとえば「HTMLファイル1つにまとめる」「日本語でコメントを書く」といったルールを Steering で伝えることで、 AIが生成するコードの品質が格段に上がります。

12

Steeringファイルを作成

左サイドバーで Kiroアイコン をクリックし、「AGENT STEERING」 セクションの 「+」 ボタンをクリック

選択肢が表示されます：
Create Steering or Import Skills - Select a method

今回は 「Project steering files」（Automatically create project steering files that help establish core workspace context）を選択してください。

左サイドバーにあるKiroのアイコン（丸い顔のマーク）をクリックすると、Kiroの機能パネルが表示されます。 その中に「AGENT STEERING」セクションがあります。 「Project steering files」を選ぶと、Kiroがプロジェクトを分析して自動でSteeringファイルを生成してくれます。

13

プロジェクトルールを記述

product.md を選択し、以下の内容をコピーして貼り付けてください：

# TODOアプリ プロジェクトガイドライン ## 技術スタック - HTML5 - CSS3 - JavaScript（ES6+） - フレームワーク・ライブラリ不要 ## コーディング規約 - index.html に HTML・CSS・JS をまとめる - const / let を使用（var 禁止） - 日本語コメントを推奨 - セマンティックなHTML要素を使用 ## 禁止事項 - var の使用禁止 - 外部ライブラリの使用禁止 - インラインスタイル禁止（style要素にまとめる）

この設定により、Kiroは今後このプロジェクトでコードを生成するとき、上記のルールに従います。 たとえば「1ファイルにまとめる」「日本語コメントで書く」といったことを自動で守ってくれます。

ℹ️
Kiroは product.md の他にも、プロジェクトのフォルダ構成を記述する structure.md や 使用する技術スタックを定義する tech.md を自動生成することがあります。 これはKiroがプロジェクト全体を理解し、一貫性のあるコードを生成するためです。 それぞれの内容も確認してみましょう。
Step 4: タスクに従って実装 ⏱ 約10分

Step 2で生成された tasks.md には、アプリを完成させるための手順が順番に並んでいます。 Kiroはこのタスクリストに沿って、1つずつコードを自動生成してくれます。 今回はHTML・CSS・JavaScriptだけなので、生成されるファイルもシンプルです。

10

タスクリストを開いて実行

生成された tasks.md を開き、Chatで以下を指示：

📝 入力するプロンプト
最初のタスクから順番に実行してください

または、タスクを選択して 「Start task」 をクリック

Kiroがタスクを1つずつ実行し、必要なファイルを自動で作成・編集していきます。 各タスクが完了するたびに、生成されたコードを確認できます。わからないことがあればChatで質問できます。

11

生成されるコードを確認

Kiroが生成するコードの例を紹介します。今回は index.html 1ファイルにすべてまとまります。実際の出力は異なる場合があります。

HTML構造（index.html の body 部分）

<div class="app"> <h1>📝 TODOアプリ</h1> <form id="todo-form" class="todo-form"> <input type="text" id="todo-input" placeholder="新しいタスクを入力..." class="todo-input" /> <button type="submit" class="add-btn">追加</button> </form> <ul id="todo-list" class="todo-list"></ul> <p id="todo-count" class="todo-count"></p> </div>

HTMLでアプリの「骨組み」を作ります。 id 属性をつけておくと、JavaScriptから要素を操作できます。 フォーム、入力欄、リスト、カウンター表示がそれぞれの役割を持っています。

CSSスタイル（index.html の style 部分）

.app { max-width: 500px; margin: 40px auto; padding: 20px; font-family: sans-serif; } .todo-form { display: flex; gap: 8px; margin-bottom: 20px; } .todo-input { flex: 1; padding: 10px; border: 2px solid #e2e8f0; border-radius: 8px; font-size: 1rem; } .add-btn { padding: 10px 20px; background: #3b82f6; color: white; border: none; border-radius: 8px; cursor: pointer; } .todo-list { list-style: none; padding: 0; } .todo-list li { display: flex; align-items: center; gap: 12px; padding: 12px; border-bottom: 1px solid #e2e8f0; } .todo-list li.completed span { text-decoration: line-through; color: #94a3b8; }

CSSでアプリの「見た目」を整えます。 display: flex でフォームの入力欄とボタンを横並びにし、 .completed クラスで完了済みタスクに取り消し線をつけています。

JavaScriptロジック（index.html の script 部分）

// ローカルストレージからTODOを読み込み let todos = JSON.parse(localStorage.getItem('todos') || '[]'); // DOM要素の取得 const form = document.getElementById('todo-form'); const input = document.getElementById('todo-input'); const list = document.getElementById('todo-list'); const count = document.getElementById('todo-count'); // TODOリストを画面に描画する関数 function renderTodos() { list.innerHTML = ''; todos.forEach(todo => { const li = document.createElement('li'); if (todo.completed) li.classList.add('completed'); li.innerHTML = ` <input type="checkbox" ${todo.completed ? 'checked' : ''}> <span>${todo.text}</span> <button class="delete-btn">🗑️</button> `; // チェックボックスで完了切り替え li.querySelector('input').addEventListener('change', () => { todo.completed = !todo.completed; saveTodos(); renderTodos(); }); // 削除ボタン li.querySelector('.delete-btn').addEventListener('click', () => { todos = todos.filter(t => t.id !== todo.id); saveTodos(); renderTodos(); }); list.appendChild(li); }); // 残りタスク数を更新 const remaining = todos.filter(t => !t.completed).length; count.textContent = `残りタスク: ${remaining}件`; } // ローカルストレージに保存 function saveTodos() { localStorage.setItem('todos', JSON.stringify(todos)); } // フォーム送信でTODO追加 form.addEventListener('submit', (e) => { e.preventDefault(); const text = input.value.trim(); if (!text) return; todos.push({ id: Date.now().toString(), text: text, completed: false }); saveTodos(); renderTodos(); input.value = ''; }); // 初期描画 renderTodos();

これがアプリの「動き」を担当するJavaScriptです。 Reactのような特別なライブラリは使わず、ブラウザ標準の機能だけで実装しています。 document.getElementById でHTML要素を取得し、 addEventListener でクリックやフォーム送信に反応させています。

Step 5: Hooksを設定（オプション） ⏱ 約3分

Hooks を使うと、「ファイルを保存したら自動でテストを実行する」といった自動化ができます。 手動で毎回テストを実行する手間が省け、バグの早期発見にもつながります。

10

自動テスト生成Hookを作成

左サイドバーの 「AGENT HOOKS」 セクションで 「+」 ボタンをクリック

設定項目値
EventFile Saved
File path*.html, *.js, *.css
📝 Instructions
ファイルが保存されたら：
1. 変更内容を分析
2. 対応するテストファイルを作成・更新
3. テストを実行して結果を報告

*.html, *.js, *.css は「プロジェクト内の全ての HTML・JS・CSS ファイル」を意味します。 このHookを設定すると、コードを保存するたびにKiroが自動でテストを作成・実行してくれます。

11

README自動更新Hookを作成（オプション）

設定項目値
EventFile Saved
File pathindex.html
📝 Instructions
HTMLファイルが変更されたら：
1. 変更内容を分析
2. README.mdの機能一覧を更新
3. 使い方の説明を追加
ℹ️
Hooksは自動で実行されるため、頻繁に変更するファイルに設定する場合は注意してください。
Step 6: 動作確認 ⏱ 約5分
12

ファイルをブラウザで開く

生成された index.html をブラウザで開きます。方法は2つあります：

方法1: ファイルをダブルクリック

方法2: ブラウザにドラッグ＆ドロップ

HTMLファイルをブラウザで直接開くだけで動作します。 これがHTML+CSS+JavaScriptだけで作るメリットです。

13

コードを変更して確認

Kiroでコードを編集したら、ブラウザで index.html をリロード（F5 または Ctrl+R）して変更を確認します。

開発サーバーは不要です。ファイルを保存してブラウザをリロードするだけで変更が反映されます。 シンプルですが、これがWeb開発の基本です。

14

機能をテスト

以下の操作ができることを確認してください：

もしアプリが表示されない場合は、ブラウザの開発者ツール（F12）でコンソールにエラーが出ていないか確認してください。 よくある原因はHTMLの閉じタグの不足やJavaScriptの構文エラーです。

🔧 困ったときは

うまくいかないときは、以下を確認してみてください。

❌ Specsが生成されない

プロンプトが短すぎるか、具体性が不足している可能性があります。

解決方法：
  • より具体的な要件を記述する（箇条書きがおすすめ）
  • 技術スタック（HTML、CSS、JavaScriptなど）を明示する
  • インターネット接続を確認する
❌ ブラウザで開いても何も表示されない

HTMLの構造にエラーがある可能性があります。

解決方法：
  • ブラウザの開発者ツール（F12）→ Console タブでエラーを確認
  • HTMLの閉じタグが正しいか確認する
  • Chatで「HTMLのエラーを修正して」と依頼する
❌ JavaScriptエラーが発生する

変数名の間違いや構文エラーの可能性があります。

解決方法：
  • ブラウザの開発者ツール（F12）→ Console タブでエラーメッセージを確認
  • Chatで「JavaScriptのエラーを修正して」と依頼する
  • scriptタグがbodyの閉じタグの直前にあるか確認する
❌ ページをリロードするとデータが消える

ローカルストレージへの保存処理が正しく動いていない可能性があります。

解決方法：
  • ブラウザの開発者ツール → Application → Local Storage で確認
  • シークレットモードでは保存されない場合があります
  • Chatで「ローカルストレージの保存が動いていません」と相談する
